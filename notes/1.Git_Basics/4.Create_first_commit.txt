4. Первый коммит.

Команда git commit фиксирует текущие изменения, добавленные в индекс, и сохраняет их в истории репозитория в виде
нового коммита.

# посмотреть состояние файлов в индексе
git status

# посмотреть историю коммитов
git log

# вывести на экран содержимое указанного файла
cat index.html

# добавить в индекс указанный файл
git add index.html

# сделать коммит (требует ввода текстового сообщения)
git commit -m "Added index.html file"

4. 2. Первый коммит: Вторая часть

Команда git add добавляет изменения из рабочего каталога в индекс (staging area) для подготовки их к последующему
коммиту.

# вывод списка настроек Git
git config --list

# установить редактор по умолчанию для Git (например, WebStorm или VS Code)
git config --global core.editor WebStorm
git config --global core.editor "code --wait"

# добавить в индекс все файлы в текущей папке
git add .

# добавить в индекс указанный файл
git add index.html

# посмотреть историю коммитов в компактном виде (только хеш и сообщение коммита)
git log --oneline
