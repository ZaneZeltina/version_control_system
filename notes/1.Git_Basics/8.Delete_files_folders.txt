8. Удаление файлов.

Команда git rm удаляет файлы из рабочего каталога и одновременно из индекса Git, подготавливая это изменение для
следующего коммита.

# вызов справки
git rm --help

# удаление файла из каталога (команда терминала)
rm about.html

# удаление файла из Git-репозитория
git rm about.html

# форсированное удаление файла (если есть изменения в индексе)
git rm about.html -f
git rm about.html --force

# удаление каталога
git rm my_folder -r
git rm my_folder --recursive

8. 2. Удаление файлов: Вторая часть.

Команда git clean удаляет неотслеживаемые файлы из рабочего каталога.

# вызов справки
git clean --help

# сценарий 1
# создадим ненужные файлы
touch some_binary_file{1..9}.bin
ls

# посмотрим их статус
git status

# очистим Git от этих файлов
git clean -f
ls
git status

# сценарий 2
# создадим пустые папки
mkdir bin
mkdir tmp
ls

# посмотрим их статус
git status

# очистим Git от этих пустых каталогов
git clean -d -f
ls
git status

# ключ -n -- 'холостой прогон команды'
# покажет, что будет происходить при выполнении команды
git clean -f -n
git clean -f -d -n

# ключ -q -- тихое выполнение команды

# не выводит комментарии о ходе своего выполнения
git clean -f -d -q

Флаг -f (force) является обязательным для подтверждения действия команды git clean — это механизм безопасности,
предотвращающий случайное удаление файлов.
